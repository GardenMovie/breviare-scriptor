# Post 3b: v0 is live — paste a URL, get a short one, click it

**Angle:** Something tangible after the planning posts. Proof that the decisions are becoming real.

**Position in series:** Between post 3 (short codes) and post 4 (redirects).

---

Brevia v0 is live.

No accounts, no dashboard, no analytics. You paste a URL, you get a short link, you click it and it redirects. That's it.

It took a few evenings. Most of that time wasn't writing code — it was making the decisions I'd already written down actually work together. The short code format I landed on (six letters, shown as XXX-XXX) went straight in. The redirect behavior I'd already decided on. Turns out planning ahead actually saves time, which I knew in theory but now know from experience.

The site is intentionally ugly. I wanted to prove the core loop worked before touching anything else. A URL goes in, a short URL comes out, the redirect works. Everything else is a feature I haven't built yet.

This is also the version I'll break things in. Better to figure out what's wrong with a barebones v0 than discover it after building something polished on top.
