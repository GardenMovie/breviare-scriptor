# Post 1: Why I'm building a URL shortener

**Angle:** "It's been done before" is exactly the point.

---

I'm building a URL shortener. Yes, Bitly exists. Yes, TinyURL exists.

That's kind of the point.

When you're early in your career, it's tempting to wait for the "original idea" before starting a real project. But original ideas are hard. Learning to engineer something well is harder. A URL shortener is small enough to finish, complex enough to teach you things, and familiar enough that you can tell when you're doing it wrong.

Brevia is my attempt to build something boring — and build it properly. Full docs before any code. Real decisions with trade-offs written down. A system that could actually run in production someday.

I'll be posting about the specific decisions I made along the way. Not code dumps — just the thinking behind it.

If you're also figuring out how to build a portfolio that shows more than syntax knowledge, follow along.
